User Mode Instructions

Creating Nodes:

  • Click anywhere on the canvas to create a node
  • Nodes are automatically labeled A-Z
  • Drag nodes to reposition them

Creating Edges:

  • Click first node - it will highlight
  • Click second node - weight input appears
  • Enter positive weight and confirm
  • Edge automatically connects the nodes

Editing & Deleting:

  • Edit Weight: Click "Edit Weight" then click any edge
  • Delete Edge: Click "Delete Edge" then click target edge
  • Delete Node: Click "Delete Node" then click target node (removes connected edges too)

Visualization:

  • Create at least 2 nodes with edges
  • Click "Visualize" to see algorithm steps
  • Use animation controls to pause/step through

Prim's Algorithm

A greedy algorithm that builds the Minimum Spanning Tree (MST) by starting from an arbitrary node and repeatedly adding the shortest edge that connects a node in the MST to a node outside it.

Key Steps:

  • Start with any node as the initial MST
  • Maintain a priority queue of edges connecting MST nodes to non-MST nodes
  • At each step, add the minimum-weight edge from the priority queue
  • Stop when all nodes are included in the MST
  • Skip the step when a cycle is encountered

Algorithm Properties:

  • Time Complexity: O(E log V) using binary heaps - Priority Queue
  • Space Complexity: O(V + E)

  • Data Structures: Priority Queue (Min-Heap)

Real-World Applications:

  • Network design (telecommunications, roads)
  • Cluster analysis
  • Image segmentation
  • Circuit design

Kruskal's Algorithm

A greedy algorithm that builds the MST by sorting all edges by weight and adding them in ascending order, skipping any edge that would form a cycle.

Key Steps:

  • Sort all edges in non-decreasing order of weight
  • Initialize a forest (each node as separate tree)
  • Iterate through sorted edges, adding each edge if it connects different trees
  • Stop when V-1 edges are added (complete MST)

Algorithm Properties:

  • Time Complexity: O(E log E)
  • Space Complexity: O(V + E)
  • Best For: Sparse graphs (when E â‰ˆ V)
  • Data Structures: Disjoint Set (Union-Find)

Real-World Applications:

  • LAN network design
  • Electric grid construction
  • Airline route planning
  • Water supply networks
